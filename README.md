
# Fighter G-Force Cockpit UI Simulator

戦闘機パイロットが急旋回時に直面する高重力加速度（G-Force）と、それに伴う生理学的限界（視覚喪失・意識喪失）をリアルタイムに計算・再現するHTML5 Canvasシミュレーターです。

純粋なJavaScript/TypeScriptロジックとWebブラウザの描画機能（Canvas API）のみで構成されており、依存ライブラリなしで動作します。

## 🚀 主な機能と再現エフェクト

### 1. リアルタイム物理・生理学シミュレーション
- **Gフォース計算**: 戦闘機の推進速度（km/h）と旋回半径（m）から、パイロットにかかる遠心力と地球重力を合成した正確なG値をリアルタイムに算出します。
- **脳内血流リザーブ**: 人間が耐G限界（耐Gスーツ着用時で約7.5G）を超えた際の、脳内酸素・血液の減少ラグ（約3〜5秒の猶予）をモデリングしています。

### 2. コックピットUI & 視覚損失エフェクト
パイロットの血流低下ステータスに応じ、画面の描画（コックピットからの視界）がリアルタイムに変化します。

* **NORMAL (正常)**: 緑を基調としたヘッドアップディスプレイ（HUD）とクリアな視界。
* **GRAY_OUT (グレーアウト)**: 血流リザーブが70%未満になると作動。画面外側から中心に向かって段階的に視野が狭まる「トンネル視界（周辺視野の喪失）」を放射状グラデーションで再現。
* **BLACK_OUT (ブラックアウト)**: リザーブが25%未満で視界の大半が闇に包まれ、中心部にわずかな光の残滓のみが残る状態。
* **G_LOC (意識喪失)**: リザーブが0%になると画面が完全に暗転（シャットダウン）。コックピットの警告音をイメージした赤文字の `!!! WAKE UP !!!` が画面中央に点滅します。

## 🛠️ 操作方法

ブラウザ画面を開いた状態で、キーボードの矢印キーを使って戦闘機を操縦します。

| キー入力 | アクション | シミュレーションへの影響 |
| :--- | :--- | :--- |
| <kbd>▲</kbd> **Up** | 加速 (最大 1500 km/h) | 速度が上がるほど、旋回時のGが急激に高くなります。 |
| <kbd>▼</kbd> **Down** | 減速 (最低 300 km/h) | 減速することでGを逃がし、血流を回復させることができます。 |
| <kbd>◀</kbd> **Left** | 旋回半径を小さくする | 旋回をタイト（急旋回）にします。**G値が跳ね上がる主因**です。 |
| <kbd>▶</kbd> **Right** | 旋回半径を大きくする | 旋回を緩やかに（直線飛行に近く）します。 |

## 🔬 実装ロジックの解説

### 遠心加速度の計算
戦闘機の速度を $v$ (m/s)、旋回半径を $R$ (m)、地球の重力加速度を $g$ ($9.80665 m/s^2$) としたとき、パイロットにかかる総G力（$G$）は以下の三平方の定理から算出されます。

$$G = \frac{\sqrt{g^2 + (\frac{v^2}{R})^2}}{g}$$

### 生理的リカバリー
限界Gを下回る操縦（減速、または旋回半径の拡大）を行うと、脳への血流が毎秒一定レートで回復し、`G_LOC` ➡️ `BLACK_OUT` ➡️ `GRAY_OUT` ➡️ `NORMAL` へと人間の意識が戻るフェーズラグをシミュレートしています。
